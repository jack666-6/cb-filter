# CB 籌碼分析平台 — 雲端部署說明

## 整體架構

```
GitHub (程式碼)
    ↓ 推送
Render.com (Flask 網頁伺服器)  ←──→  Supabase (PostgreSQL 資料庫)
                                           ↑
                                    GitHub Actions
                                    (每天 18:30 自動更新)
```

---

## 步驟一：建立 Supabase 資料庫（免費）

1. 前往 https://supabase.com → **Start your project**
2. 用 GitHub 帳號登入
3. 點 **New project**
4. 填入：
   - Project name：`cb-analysis`
   - Database Password：自訂一組強密碼（**記下來！**）
   - Region：**Southeast Asia (Singapore)**（離台灣最近）
5. 等待建立（約 1 分鐘）
6. 建立完成後，點左側 **Project Settings → Database**
7. 找到 **Connection string → URI** 那一行（格式如下）：
   ```
   postgresql://postgres:[YOUR-PASSWORD]@db.xxxx.supabase.co:5432/postgres
   ```
   **複製這個字串，後面會用到。**

---

## 步驟二：建立 GitHub Repository

1. 前往 https://github.com → **New repository**
2. Repository name：`cb-analysis`
3. 選 **Private**（避免 Token 外洩）
4. **不要**勾選 Initialize README
5. 點 **Create repository**

### 把 cloud_deploy 資料夾上傳到 GitHub

在你的電腦上打開 **命令提示字元（CMD）**，執行：

```cmd
cd C:\Users\User\OneDrive\桌面\程式\cloud_deploy

git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的帳號/cb-analysis.git
git push -u origin main
```

---

## 步驟三：設定 GitHub Secrets

1. 進入你的 GitHub Repository → **Settings → Secrets and variables → Actions**
2. 點 **New repository secret**，新增以下兩個：

| Secret 名稱 | 值 |
|------------|-----|
| `DATABASE_URL` | 步驟一複製的 Supabase URI（把 `postgres://` 改成 `postgresql://`） |
| `FINMIND_TOKEN` | `Bearer eyJ0eXAiOiJKV1Q...`（你的 FinMind Token，含 Bearer 前綴） |

---

## 步驟四：初始化資料庫（搬移本機資料）

在你的電腦上（有 `market_data.db` 的地方），打開 CMD：

```cmd
cd C:\Users\User\OneDrive\桌面\程式\cloud_deploy

pip install psycopg2-binary

set DATABASE_URL=postgresql://postgres:你的密碼@db.xxxx.supabase.co:5432/postgres

python migrate_sqlite_to_pg.py
```

> 這會把你本機所有歷史資料一次性搬到 Supabase，約需 5~30 分鐘。

---

## 步驟五：部署到 Render.com（免費）

1. 前往 https://render.com → 用 GitHub 帳號登入
2. 點 **New → Web Service**
3. 連接你的 GitHub Repository `cb-analysis`
4. 填入設定：

| 欄位 | 值 |
|------|-----|
| Name | `cb-analysis` |
| Runtime | `Python 3` |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `gunicorn app:app --workers 2 --bind 0.0.0.0:$PORT --timeout 120` |
| Instance Type | **Free**（或 Starter $7/月，不會休眠） |

5. 往下找 **Environment Variables**，新增：

| Key | Value |
|-----|-------|
| `DATABASE_URL` | 你的 Supabase URI |
| `FINMIND_TOKEN` | 你的 FinMind Token |

6. 點 **Create Web Service**
7. 等待部署完成（約 3~5 分鐘）
8. 部署完成後，Render 會給你一個網址，例如：
   ```
   https://cb-analysis.onrender.com
   ```
   **這就是你的網頁網址，可以分享給任何人！**

---

## 步驟六：確認每日自動更新

1. 進入 GitHub Repository → **Actions** 頁面
2. 你會看到 `每日資料更新` 工作流程
3. 每天台灣時間 **18:30**（週一到週五）自動執行
4. 也可以點 **Run workflow** 手動觸發

---

## 免費方案限制說明

### Render.com 免費版
- ✅ 完全免費
- ⚠️ **閒置 15 分鐘後會休眠**，第一次打開需等 30~60 秒啟動
- 💡 解決方法：用 [UptimeRobot](https://uptimerobot.com) 免費監控，每 5 分鐘 ping 一次 `/health`，讓它不休眠

### Supabase 免費版
- ✅ PostgreSQL 500MB 儲存空間
- ✅ 無流量限制
- ⚠️ 超過 7 天沒有 API 連線會暫停（每週至少用一次就不會）

### GitHub Actions 免費版
- ✅ 每月 2000 分鐘（每次更新約 10 分鐘，一個月最多 200 次，完全足夠）

---

## UptimeRobot 設定（避免 Render 休眠）

1. 前往 https://uptimerobot.com → 免費註冊
2. **Add New Monitor**
3. 設定：
   - Monitor Type：`HTTP(s)`
   - URL：`https://cb-analysis.onrender.com/health`
   - Monitoring Interval：`5 minutes`
4. 儲存後，你的網頁就會 24 小時保持在線！

## 檔案結構說明

```
cloud_deploy/
├── .github/
│   └── workflows/
│       └── daily_update.yml     ← GitHub Actions 每日自動更新
├── templates/
│   └── index.html               ← 前端網頁（不用改）
├── app.py                       ← Flask 網頁伺服器（PostgreSQL 版）
├── config.py                    ← 設定（從環境變數讀取）
├── data_fetcher.py              ← FinMind API 資料抓取
├── db_manager_pg.py             ← PostgreSQL 資料庫操作
├── db_updater.py                ← 每日更新主程式
├── migrate_sqlite_to_pg.py      ← 一次性資料移轉工具
├── requirements.txt             ← Python 套件清單
└── Procfile                     ← Render 啟動設定
```
